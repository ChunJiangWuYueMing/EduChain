# EduChain 答辩演示脚本

## 总时长：8-10 分钟

---

## 第一步：启动系统（提前准备，答辩时不演示）

```bash
# 1. 安装依赖
npm install
pip install -r requirements.txt

# 2. 编译合约
npm run compile

# 3. 启动 Docker（Ganache + 后端 + 前端）
docker compose up --build

# 4. 新开终端，部署合约
python3 scripts/deploy.py

# 5. 确认服务正常
curl http://localhost:5000/api/health
```

访问 `http://localhost:8080` 进入系统。

## 第六步：验证原始文件（1 分钟）

1. 切换到"真伪验证"页面。
2. 输入刚才上传的资料 ID（可从市场详情获取）。
3. 选择文件 `demo/original_区块链技术课件.pptx`。
4. 点击"开始验证"。
5. **展示效果**：
   - SHA-256 精确校验：✓ 完全一致（绿色）
   - 综合判定：✓ 文件完整可信
   - 内容相似度：100%
   - 汉明距离：0 / 256
   - 分类：identical（完全一致）
6. **说明**：SHA-256 和 SimHash 都是 256 位的，SHA-256 负责比特级完整性，SimHash 负责内容相似度。

---

## 第七步：验证被篡改的 PPT ⭐核心亮点⭐（2 分钟）

1. 在验证页面，选择文件 `demo/tampered_区块链技术课件.pptx`（修改版）。
2. 资料 ID 保持不变。
3. 点击"开始验证"。
4. **展示效果**：
   - SHA-256 精确校验：✗ 不一致（红色）← 注意：SHA-256 完全不同
   - 综合判定：⚠ 文件已被篡改
   - **内容相似度：约 75-85%**（大字醒目，黄色）
   - 汉明距离：约 35-50 / 256
   - 分类：**derived（衍生版本 / 疑似篡改）**
   - **关键词差异**：
     - 共同关键词：区块链、智能合约、以太坊...
     - 新增关键词：高性能、企业级、Solana...
     - 缺失关键词：透明性、Solidity...
5. **重点说明**：
   - PPT 修改了 2 页文字后重新保存，SHA-256 完全不同（因为文件字节彻底变了）。
   - 但 SimHash 内容指纹仍然能识别出它们高度相似——这就是"针对内容的防篡改"。
   - SHA-256 告诉你"文件不一样了"，SimHash 告诉你"改了多少、是不是同一个来源"。
   - 两者配合，既保证了精确校验，又实现了内容级相似度检测。

## 总结陈述（0.5 分钟）

EduChain 实现了校园学习资料共享的三大核心能力：

1. **文件验真**：SHA-256 精确哈希 + 256 位 SimHash 内容指纹，既能检测文件是否被修改，也能量化内容的相似程度。
2. **权限控制**：三种访问策略（公开 / 同课程 / 白名单），策略数据上链可审计。
3. **通证激励**：注册 +100、上传 +20、下载付费转移，形成资源交换闭环。

技术栈：Solidity 智能合约 + Python Flask 后端 + Vanilla JS 前端 + Ganache 本地测试链。
