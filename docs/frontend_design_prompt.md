# EduChain 前端高保真设计文档

本文档用于指导生图 AI 生成 EduChain 前端页面高保真设计图。设计目标不是营销落地页，而是一个融合西南交通大学校园气质的高效、清晰、可信的学习资料区块链分发系统工作台。

## 1. 产品定位

EduChain 是面向校园学习资料共享的区块链可信分发原型系统。前端需要让用户完成登录、浏览资料、上传资料、下载付费、文件验证、钱包查询和审计追溯。

核心感受：

- 可信：强调链上存证、哈希、SimHash、交易哈希、下载日志。
- 高效：页面以任务流为中心，减少装饰性内容。
- 学术/校园：融合西南交通大学的 Logo 识别、交大蓝主色和校园学术气质，视觉专业、干净、有归属感。
- 原型系统：保留必要技术信息，但用结构化 UI 展示，避免大段 JSON。
- 适度自由：允许更有设计感的首页构图、品牌头图、校园线条纹理和模块化仪表盘，但不能牺牲任务效率。

## 2. 品牌融合方向

本系统由西南交通大学学生完成，视觉上应体现“交大出品”的身份感，但不要做成学校官网或招生宣传页。

品牌元素使用建议：

- 在登录页、侧边栏、顶部栏或页面角落融合西南交通大学 Logo。
- 如果可以使用真实 Logo，请保持 Logo 清晰、比例正确、留白充足。
- 如果生图 AI 无法准确还原 Logo，可使用“西南交通大学 Logo 占位区域”或抽象化校徽容器，不要生成错误文字或变形校徽。
- 主色使用交大蓝方向。建议从用户提供的校徽或学校视觉素材中取样；如无取样，可使用深校蓝作为主色，例如 `#005BAC`、`#0067B1` 或接近校徽蓝的稳重蓝色。
- 可少量使用白色、浅灰、浅蓝和金色/暖黄作为辅助色，表达校园荣誉感，但金色只能点缀，不能喧宾夺主。
- 可以引入与交通大学相关的抽象视觉语言：轨道线、桥梁线条、流动路径、知识网络、链上节点。
- 区块链元素应克制表达，使用节点、链路、存证标签即可，不要做夸张币圈风格。

推荐品牌气质关键词：

- 交大蓝
- 校园可信平台
- 学术资料共享
- 轨道交通线条感
- 链上存证
- 清爽、可靠、秩序感

避免：

- 不要把页面做成学校官网首页。
- 不要使用大面积红黄校庆风。
- 不要把 Logo 当作背景水印铺满页面。
- 不要生成错误校名、错误校徽文字或模糊标识。

## 3. 整体信息架构

建议采用单页应用风格的后台工作台布局，即使实现上是原生 HTML/CSS/JS，也应按统一导航组织页面。

一级导航：

1. 资料市场
2. 上传资料
3. 文件验证
4. 我的钱包
5. 审计追溯
6. 系统状态

登录态：

- 未登录时显示登录页。
- 登录后进入资料市场。
- 顶部右侧显示用户姓名、学号、EDU 余额、简短 ETH 地址和登出按钮。

页面数量建议：

- 登录页
- 资料市场页
- 资料详情面板或详情页
- 上传资料页
- 文件验证页
- 我的钱包页
- 审计追溯页
- 系统状态页

## 4. 统一布局规范

### 桌面端

- 画布建议：1440 x 1024。
- 布局可以自由一些，不强制固定左侧侧边栏。
- 可选布局 A：左侧品牌导航 + 顶部用户栏 + 主内容区。
- 可选布局 B：顶部横向导航 + 左侧轻量筛选区 + 主内容区。
- 可选布局 C：校园品牌化仪表盘首页 + 页面内 Tabs + 详情抽屉。
- 主内容区使用 24px 到 32px 外边距。
- 页面主体采用 12 栅格、两栏布局或明确的主次分区。
- 重要操作靠近标题区右侧，如“上传资料”“刷新”“导出”等。
- 资料列表、交易记录、审计记录仍应以表格为核心，不要为了自由构图牺牲可读性。

### 移动端

- 画布建议：390 x 844。
- 侧边栏折叠为底部导航或顶部菜单按钮。
- 表格改为卡片列表。
- 长哈希、地址、交易哈希必须自动截断，中间省略。
- 主要按钮宽度适合拇指点击，高度不低于 40px。

### 顶部栏

必要元素：

- 当前页面标题。
- 全局搜索入口，可选，仅在资料市场和审计页明显展示。
- 后端/链状态小指示点：正常、异常、检查中。
- 用户信息区域：姓名、学号、EDU 余额、短地址。
- 登出图标按钮。
- 可加入小尺寸西南交通大学 Logo 或“SWJTU / EduChain”联合标识。

### 导航区

必要元素：

- 联合品牌：西南交通大学 Logo + EduChain。
- 副标题：可信资料交换，或“校园学习资料可信分发”。
- 导航项：资料市场、上传资料、文件验证、我的钱包、审计追溯、系统状态。
- 当前页面高亮。
- 底部显示当前网络：Ganache Local，配小圆点状态。
- 导航可以是左侧栏、顶部导航、分段按钮或移动端底部导航。

避免：

- 不做纯营销式大面积英雄区；登录页和首页可以有适度品牌视觉区域。
- 不做营销口号卡片。
- 不做过多渐变背景；允许少量交大蓝渐变用于品牌头部或重点按钮。
- 不把每个小指标都做成巨大卡片。

## 5. 视觉风格

### 色彩

推荐主色：

- 主色：交大蓝方向，建议 `#005BAC`、`#0067B1` 或从真实 Logo 取样的深蓝。
- 主色深阶：#003F7D，用于导航深色底、选中态、标题强调。
- 主色浅阶：#EAF3FF，用于浅蓝背景、筛选区、信息提示。
- 点缀色：#D6A84F 或低饱和金色，仅用于学校荣誉感、小徽章、重点数据点缀。
- 成功：#16A34A。
- 警告：#D97706。
- 危险：#DC2626。
- 中性背景：#F6F8FB。
- 页面白底：#FFFFFF。
- 主文字：#111827。
- 次文字：#6B7280。
- 边框：#E5E7EB。

限制：

- 不要使用大面积紫蓝赛博渐变。
- 不要做 Web3 霓虹黑金风。
- 不要使用过多玻璃拟态。
- 不要一屏全是蓝色，要用白、灰、浅蓝、少量金色和状态色保持信息密度。
- 品牌蓝应服务于导航、重点按钮、状态表达和页面识别，不要覆盖所有卡片。

### 字体

- 中文使用系统无衬线字体，如 PingFang SC、Microsoft YaHei、Noto Sans SC。
- 英文和数字使用 Inter 或系统无衬线。
- 哈希、地址、交易哈希使用等宽字体。

字号建议：

- 页面标题：24px。
- 区块标题：18px。
- 表格/表单主文字：14px。
- 辅助说明：12px。
- 按钮文字：14px。

### 组件样式

- 卡片圆角 8px。
- 按钮圆角 6px。
- 输入框圆角 6px。
- 表格行高 48px 左右。
- 阴影非常轻，只用于浮层、详情抽屉、弹窗。
- 使用清晰分割线，而不是大量阴影。
- 可在顶部区域、登录页背景、空状态中加入细线轨道纹理、节点连线、桥梁线条或校园蓝图式图形。
- 纹理必须很淡，不要干扰表格和表单阅读。

图标建议：

- 资料市场：文件夹或书本图标。
- 上传资料：上传图标。
- 文件验证：盾牌检查图标。
- 我的钱包：钱包图标。
- 审计追溯：时间线或列表搜索图标。
- 系统状态：活动脉冲或服务器图标。

## 6. 通用交互状态

每个页面都必须设计以下状态：

- Loading：骨架屏或局部加载行，避免整页空白。
- Error：红色或琥珀色提示条，包含简短错误原因和重试按钮。
- Empty：空状态插画不宜夸张，提供下一步按钮。
- Success：上传、登录、验证完成后使用轻量成功提示。
- Disabled：权限不足、余额不足、未登录时按钮禁用并显示原因。

通用反馈：

- 下载、上传、验证、删除都是高成本动作，需要按钮 loading 状态。
- 删除资料需要确认弹窗。
- 复制地址、哈希、交易哈希需要一键复制图标和复制成功提示。
- 长字段使用中间省略，例如 `0x12ab...89ef`。

## 7. 登录页设计

页面目标：让学生快速登录系统。

必要元素：

- 居中登录面板，宽度约 400px。
- 联合标识：西南交通大学 Logo + EduChain。
- 标题：校园学习资料可信交换平台。
- 输入框：学号、密码。
- 主按钮：登录。
- 辅助信息：默认测试账号提示可以很小，不要喧宾夺主。
- 登录错误提示。
- 页面底部可显示系统状态小标签：后端连接、链连接。

登录成功后返回数据应在后续页面顶部展示：

- 姓名
- 学号
- ETH 地址
- 课程列表
- 角色
- EDU 余额
- 首次登录奖励，如有则用轻量提示展示 `+100 EDU`

视觉要求：

- 背景可以更有品牌感：交大蓝渐变顶部、抽象轨道线、校园建筑轮廓、链路节点纹理，保持克制。
- 登录面板旁可以放一个小型品牌信息区，展示“西南交通大学学生可信资料共享原型系统”。
- 不使用夸张全屏插画英雄。
- 登录面板信息密度适中。

## 8. 资料市场页

页面目标：浏览、搜索、筛选、查看和下载学习资料。

桌面布局：

- 页面标题：资料市场。
- 标题右侧主按钮：上传资料。
- 标题区可加入“SWJTU EduChain”小品牌条、交大蓝背景条或轻量校园纹理。
- 顶部筛选条：
  - 搜索框：搜索资料名或课程。
  - 课程筛选下拉。
  - 访问策略筛选，可选。
  - 刷新图标按钮。
- 主体为资料表格。
- 右侧或弹出抽屉显示资料详情。

资料表格必要列：

- 资料名称
- 课程
- 价格 EDU
- 访问策略：公开、同课程、白名单预留
- 上传者短地址
- 版本
- 上传时间
- 状态：正常、已删除
- 操作：详情、验证、下载

资料卡片移动端字段：

- 资料名称
- 课程标签
- 价格 EDU
- 访问策略
- 上传时间
- 操作按钮组

资料详情抽屉必要信息：

- 资料 ID
- 名称
- 课程
- 上传者地址
- SHA-256
- SimHash
- 文本长度
- 价格
- 版本
- 访问策略
- 链上时间
- 下载按钮
- 验证此资料按钮
- 查看下载记录按钮

交互要求：

- 点击资料行打开详情抽屉。
- 下载前检查登录态；未登录时引导登录。
- 余额不足时下载按钮禁用或显示错误提示。
- 下载自己的资料时显示不可下载提示。
- 已删除资料弱化显示，不允许下载。

不要：

- 不要用瀑布流卡片展示核心数据。
- 不要隐藏价格、课程和访问策略。
- 不要把哈希完整铺满页面，要有展开/复制能力。

## 9. 上传资料页

页面目标：上传文件并完成链上登记。

布局建议：

- 左侧主表单，右侧上传说明/结果面板。
- 表单区域不超过 720px 宽，避免横向分散。

必要表单元素：

- 文件上传区域：拖拽上传 + 点击选择。
- 支持格式提示：pdf、docx、pptx、txt、md。
- 文件大小提示：不超过 50MB。
- 课程编号输入或选择。
- 下载价格 EDU，数字输入。
- 访问策略单选：
  - 公开
  - 同课程
  - 白名单，标记为预留/暂不可用
- 提交按钮：上传并存证。

上传中状态：

- 文件指纹计算中。
- 链上登记中。
- 上传奖励确认中。
- 按步骤显示进度，不要只显示一个旋转图标。

上传成功结果面板：

- 资料 ID
- 资料名称
- SHA-256
- SimHash
- 文本长度
- 价格 EDU
- 上传奖励：+20 EDU
- 相似资料列表
- 交易哈希
- 返回市场按钮
- 继续上传按钮

相似资料列表字段：

- 资料 ID
- 名称
- 课程
- 相似度或汉明距离
- 分类：identical、high、derived、different

错误状态：

- 不支持文件格式。
- 课程编号为空。
- 价格不是整数。
- 重复资料或上传失败。

视觉要求：

- 上传区域清晰但不夸张。
- 链上登记和指纹结果用结构化信息块展示。
- 不要做成复杂向导，单页足够。

## 10. 文件验证页

页面目标：用户上传本地文件，与链上记录比对，查看完整性报告。

布局建议：

- 顶部输入资料 ID 或从资料详情带入。
- 主体分为两栏：
  - 左侧：待验证文件上传。
  - 右侧：验证报告。

必要元素：

- 资料 ID 输入框。
- 文件上传框。
- 开始验证按钮。
- 验证报告状态徽章：
  - 完全一致
  - 高度相似
  - 衍生版本
  - 差异较大
  - 疑似篡改

验证报告必要字段：

- SHA-256 是否匹配。
- 本地 SHA-256。
- 链上 SHA-256。
- 本地 SimHash。
- 链上 SimHash。
- 汉明距离。
- 相似度百分比。
- 分类。
- 是否被篡改。

视觉表达：

- 顶部给一个明确结论条，如“文件完整性通过”或“疑似篡改”。
- SHA 匹配使用绿色，失败使用红色。
- SimHash 相似度可用水平进度条或环形数值，但不要过度装饰。
- 哈希值使用等宽字体并支持复制。

空状态：

- 未选择资料和文件时，右侧显示简洁占位。

## 11. 我的钱包页

页面目标：查看 EDU 余额和链上通证交易历史。

布局建议：

- 顶部余额摘要区。
- 下方交易历史表格。

余额摘要必要元素：

- 当前 EDU 余额，大数字但不要做金融交易所风格。
- ETH 地址短显示，支持复制。
- 用户姓名和学号。
- 注册奖励说明，如已收到可显示 `注册奖励 +100 EDU`。

交易历史表格必要列：

- 类型：mint、receive、send、burn。
- 数量 EDU。
- From 地址。
- To 地址。
- 区块号。
- 交易哈希。
- 时间，如接口无法提供时间，可显示区块号为主。

交互：

- 分页。
- 刷新。
- 类型筛选。
- 复制交易哈希。

视觉要求：

- 钱包页应简洁可靠，不要做加密货币行情盘。
- 使用状态色区分收入、支出、销毁。

## 12. 审计追溯页

页面目标：查询资料下载记录、我的下载、我的上传和管理员全局日志。

建议使用页内 Tabs：

- 我的下载
- 我的上传
- 资料下载记录
- 全局审计，管理员可见

我的下载表格字段：

- 资料 ID
- 上传者地址
- 价格 EDU
- 文件哈希
- 时间

我的上传表格字段：

- 资料名称
- 课程
- 价格
- 版本
- 上传时间
- 状态
- 查看下载记录

资料下载记录查询：

- 输入资料 ID。
- 查询按钮。
- 结果表格字段：下载者、上传者、价格、文件哈希、时间。

全局审计：

- 仅管理员显示。
- 表格字段：资料 ID、下载者、上传者、价格、文件哈希、时间。
- 分页。

视觉要求：

- 审计页信息密度可以稍高。
- 使用表格优先，不使用大卡片。
- 对地址和哈希使用短显示 + 复制。

## 13. 系统状态页

页面目标：开发/演示时查看系统、后端、链和合约状态。

必要元素：

- 后端状态：running/error。
- Ganache 连接状态。
- Ganache URL。
- 当前区块号。
- Deployer 地址。
- 合约地址：
  - EduToken
  - MaterialRegistry
  - DownloadLog
- 资料总数。
- 下载记录总数。
- 刷新按钮。

布局：

- 顶部状态摘要：后端、链、合约、数据。
- 下方合约地址表。
- 错误时显示链错误信息。

视觉要求：

- 这是系统运维/演示页，不要做成主要首页。
- 技术字段可以完整但要整齐。

## 14. 组件清单

生图 AI 需要在页面中体现以下组件风格：

## 16. 高保真生图总提示词

请生成一套 EduChain 校园学习资料区块链可信分发系统的高保真 UI 设计图。系统由西南交通大学学生完成，视觉上需要融合西南交通大学 Logo 识别和交大蓝主色。它是一个专业、高效、信息密度适中的 Web 工作台，不是营销落地页，也不是学校官网。布局可以自由一些，可采用左侧导航、顶部横向导航、品牌化仪表盘或清晰的分区工作台，但必须保证资料列表、上传、验证、钱包、审计等任务高效完成。整体视觉干净、可信、有校园学术感，使用白色、浅灰、交大蓝、浅蓝和少量金色点缀，绿色/红色/琥珀色表达状态。页面包含资料市场、上传资料、文件验证、我的钱包、审计追溯、系统状态等模块。

设计必须突出真实业务元素：资料名称、课程、价格 EDU、访问策略、上传者地址、版本、SHA-256、SimHash、交易哈希、下载记录、EDU 余额。所有长地址和哈希使用等宽字体、中间省略、带复制图标。表格优先，卡片用于摘要、结果和关键状态。交互元素要完整，包括搜索、筛选、上传框、按钮、分页、详情抽屉、状态徽章、错误提示、空状态和 loading 状态。可以加入抽象轨道线、桥梁线条、链路节点、校园蓝图纹理作为轻量背景装饰。

不要生成宣传首页、不要做成学校官网、不要大面积插画、不要 Web3 霓虹风、不要复杂渐变、不要装饰性过强的元素。不要生成错误校名或变形 Logo；如无法准确还原 Logo，请使用清晰的 Logo 占位区。界面应看起来可以直接交给前端开发复刻。

## 18. 设计验收标准

高保真图生成后，应满足：

- 每个页面一眼能看出核心任务。
- 所有必要字段都有位置。
- 主操作不超过 1 个，次操作清晰排列。
- 表格、筛选、分页、状态反馈完整。
- 技术字段不会挤压主要业务信息。
- 桌面端不空旷，移动端不拥挤。
- 页面风格统一，能组成同一个产品。
- 有明确的西南交通大学学生项目归属感，但不是学校官网。
- 使用交大蓝作为主视觉识别，颜色自由但克制。
- 没有过度营销化、装饰化、重复解释性的 UI。
