# EduChain 联动测试手册

## 测试准备

1. 管理员确认服务器健康检查显示9个账号、8个学生、1个管理员和9个钱包。
2. 每人准备1至10MB、不含隐私信息的PDF或PPTX资料。
3. 唐昊分别使用普通窗口登录学生账号、无痕窗口登录管理员账号。
4. 所有人记录操作前后余额、资料ID、交易哈希和截图。

## 联动流程

1. 同步登录：8名学生应各获得一次100 EDU，管理员不得获得自动奖励。
2. 同步上传：验证正常上传、20 EDU上传奖励、SHA-256重复拦截和SimHash相似提示。
3. 权限测试：分别测试公开、同课程、无课程权限、白名单允许和白名单拒绝。
4. EDU测试：验证下载付款、上传者收入、普通转账、管理员奖励和扣罚。
5. 文件验证：原文件应通过，篡改文件应提示SHA-256不一致。
6. 权限边界：学生不能奖励、扣罚或删除他人资料；管理员可以执行管理操作。
7. 持久化：记录区块、合约、余额和资料后重启后端，再执行整套 `down` / `up`，数据应保持。
