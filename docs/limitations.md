# EduChain 当前限制说明

本文档诚实列出当前原型的已知限制，供答辩时坦诚说明。

---

## 1. 区块链环境

- 当前运行在本地 Ganache 测试链上，chainId=1337，10 个预置测试账户。
- 合约每次重新部署会获得新地址，历史数据不可跨部署持久化。
- 未接入真实的以太坊主网或测试网（如 Sepolia）。
- 交易确认依赖单节点，不存在真实的多节点共识。

## 2. 文件存储

- 文件本体存储在 `backend/uploads/` 目录下，是链下存储。
- 区块链上只记录文件的 SHA-256 哈希、SimHash 内容指纹和元数据。
- 如果 uploads/ 目录被清空，已注册资料将无法下载，但链上记录仍然可查。
- 生产环境建议使用 IPFS 或 Filecoin 等去中心化存储方案。

## 3. 权限控制

- 链上保存的是访问策略数据（policyType 和 policyValue）。
- 权限判断逻辑在后端 Flask 应用层执行（material_service._check_access_policy）。
- 区块链保证了策略数据的可审计性和难以篡改性，但这不等于完全去中心化的访问控制。
- 如果后端被攻破，攻击者可以绕过权限检查直接请求文件。

## 4. 内容指纹（SimHash）

- SimHash 是内容相似度检测算法，**不能替代人工判断**。
- 同类课程资料（如多份"数据结构"课件）共享大量专业术语，SimHash 可能误判为高相似度。
- SimHash 对文本内容的细微改写敏感度有限——如果只改了几个字，汉明距离可能仍然很小。
- 当前的文本提取依赖 python-pptx、python-docx、PyPDF2 提取文本层。
- **对扫描版 PDF（图片型）或纯图片型 PPT 的文字识别能力有限**，因为这些文件不包含可提取的文本层。生产环境需接入 OCR（如 Tesseract）。

## 5. 通证经济

- EDU 是系统内学习积分通证，不涉及真实货币，不具备任何经济价值。
- 当前经济参数（注册+100、上传+20、下载价格默认5、扣罚最多50）是预设的，未经过激励效果验证。
- decimals=0 意味着不存在小数积分，所有操作都是整数。
- 通证交易历史从 ERC-20 Transfer 事件解析。在部分 Ganache 版本上事件过滤可能不完整。

## 6. 用户管理

- 用户数据存储在 `backend/users.json` 中，是静态文件。
- 不支持用户自主注册，新增用户需要手动在 users.json 中添加条目。
- 密码使用 werkzeug 哈希存储，但用户私钥派生依赖固定助记词（仅开发环境）。
- 生产环境应使用数据库存储用户信息，并使用安全的密钥管理方案。

## 7. 合约局限

- MaterialRegistry 的 update() 是覆盖式更新（最新哈希 + 版本号递增），不保留完整的历史版本数组。
- 历史版本追加是后续优化方向。
- DownloadLog 的 recordDownload() 权限是 onlyOwner（deployer），实际由后端以 deployer 身份调用。在去中心化场景中需要调整为多方签名或事件驱动。

## 8. API 安全

- 当前没有实现 API 频率限制（rate limiting）。
- session 使用 Flask 默认的 signed cookie，SECRET_KEY 是硬编码的开发密钥。
- 文件上传大小限制为 50MB。
- CORS 对所有来源开放（开发环境）。

## 9. 前端

- 前端是纯 HTML/CSS/Vanilla JS 单页应用，没有使用任何前端框架。
- 已完成基础响应式布局，移动端可以浏览核心页面；复杂表格和答辩演示仍以桌面浏览器体验最佳。
- 未实现 Web Crypto API 的客户端二次哈希校验（规划中）。
- 下载文件的完整性校验依赖后端 SHA-256 对比，前端未独立验证。

## 10. 文档格式支持

- 当前支持：PPTX（python-pptx）、DOCX（python-docx）、PDF（PyPDF2）、TXT、MD。
- 不支持：旧版 PPT（.ppt）、旧版 DOC（.doc）、XLS/XLSX、图片格式。
- PPTX 提取仅获取 shape 中的文本和表格，不提取图表、SmartArt 等复杂元素。
- PDF 中嵌入的图片文字无法识别。

---

## 答辩建议

在答辩中应主动提及上述限制，体现对系统边界的清醒认识，这比回避问题更能获得老师的认可。重点强调：

1. 本系统是**原型**，聚焦验证"内容指纹 + 通证激励"的可行性。
2. SimHash 是**辅助检测**工具，不是绝对防篡改方案。
3. 权限在**后端执行**，但策略数据在链上**可审计**。
4. EDU 是**学习积分**，不是加密货币。
