## 主要问题

1. **严重：扣款与下载结果不一致**

   方天宇下载资料G时被扣除5 EDU，谢傲宇收到5 EDU，但接口返回nonce错误，没有返回文件，也没有生成DownloadLog记录。

2. **并发下载稳定性不足**

   9个下载请求并发时出现nonce冲突、Ganache 10秒读取超时和短暂502。降低并发后，资料E可以成功下载。

3. **李子彤下载资料A持续超时**

   并发测试和后续单请求补测均返回Ganache读取超时。李子彤余额仍为120 EDU，allowance为0，本次没有扣款。

4. **管理员不能删除其他用户资料**

   后端允许管理员进入删除逻辑，但智能合约仍要求调用者必须等于上传者，因此管理员删除资料H失败。

5. **资料更新没有持久化**

   周子皓修改资料E时接口返回成功，但再次查询仍显示旧名称。

6. **钱包交易历史为空**

   转账和下载支付已经发生，但方天宇、唐昊的 `/api/token/history` 均返回0条记录。

7. **下载审计不完整**

   已发生5笔下载付款，但仅有4条下载审计。缺失的是资料G发生扣款但接口失败的记录。
