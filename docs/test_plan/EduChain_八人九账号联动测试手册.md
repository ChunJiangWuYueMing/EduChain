# EduChain八人九账号联动测试手册

## 一、测试目标

本次测试验证：

1. 8名成员能否通过公网同时访问EduChain。
2. 9个账号能否正确登录。
3. 唐昊的学生账号和管理员账号能否正确区分。
4. 每个学生是否对应独立钱包。
5. 资料上传和链上存证是否成功。
6. SHA-256完全重复检测是否有效。
7. SimHash相似资料检测是否有效。
8. 课程访问策略是否有效。
9. 指定用户白名单策略是否有效。
10. EDU上传奖励、下载支付和普通转账是否正确。
11. 文件完整性验证是否有效。
12. 管理员奖励、扣罚和删除是否有效。
13. 多人同时操作时系统是否稳定。
14. 容器重启后数据是否保留。

本次测试属于：

```text
基于单节点Ganache测试链的多用户联动测试
```

不将其描述为多节点去中心化网络实验。

## 二、测试资料准备原则

资料由成员自行准备，优先使用PDF或PPTX。

建议：

- 单个文件1MB至10MB；
- 不包含身份证号、成绩单等隐私信息；
- PDF应尽量包含可复制文字，不要全部为扫描图片；
- PPT应包含正文段落，不要只有图片；
- 文件主题与对应课程一致；
- 文件名简洁清楚；
- 保留每个测试文件的原始副本。

## 五、正式测试流程

### 阶段1：服务器检查

管理员执行：

```bash
docker compose -f docker-compose.server.yml ps
```

确认：

- 前端正常；
- 后端正常；
- Ganache正常；
- 9个账号加载成功；
- 9个钱包加载成功；
- 三个智能合约可用。

### 阶段2：同步登录

所有成员准备登录页面，主持人倒数：

```text
3、2、1，登录
```

检查：

- 所有人登录成功；
- 页面没有串号；
- 8名学生各获得一次100 EDU；
- 管理员没有获得学生奖励；
- 重复刷新不重复奖励。

### 阶段3：同步上传

成员提前选好文件，主持人倒数后同时点击上传。

检查：

- 服务没有崩溃；
- 成功上传生成独立资料ID；
- 成功上传获得20 EDU；
- 完全重复文件被拒绝；
- 相似文件得到相似提示；
- 日志没有明显nonce冲突。

### 阶段4：权限测试

执行：

1. 公开资料下载。
2. BC401同课程下载。
3. CS301同课程下载。
4. CS302同课程下载。
5. 没有课程权限的下载。
6. 白名单允许用户下载。
7. 白名单外用户下载。

失败请求必须：

- 不扣除EDU；
- 不产生成功下载记录；
- 不返回文件；
- 给出明确原因。

### 阶段5：EDU测试

测试：

- 学生首次登录奖励；
- 上传奖励；
- 下载付款；
- 上传者收入；
- 普通用户转账；
- 管理员奖励；
- 管理员扣罚。

记录每次操作前后的余额与交易哈希。

### 阶段6：文件验证

使用资料A原文件：

```text
预期：完整性验证通过
```

使用资料I：

```text
预期：SHA-256不一致，提示文件已被修改
```

资料C：

```text
预期：不是完全相同文件，但与资料A存在较高相似性
```

### 阶段7：权限边界

验证：

- 学生不能奖励；
- 学生不能扣罚；
- 学生不能删除别人资料；
- 上传者可以管理自己的资料；
- 管理员可以删除其他用户资料；
- 管理员执行管理功能不受普通学生课程限制。

### 阶段8：持久化测试

记录：

- 当前区块高度；
- 三个合约地址；
- 资料数量；
- 各账号余额；
- 下载记录数量。

执行：

```bash
docker compose -f docker-compose.server.yml restart backend
```

验证数据不变。

随后执行：

```bash
docker compose -f docker-compose.server.yml down
docker compose -f docker-compose.server.yml up -d
```

再次验证：

- 合约地址不变；
- 钱包地址不变；
- EDU余额不变；
- 资料记录不变；
- 上传文件仍可下载；
- 登录仍然有效。

不得执行：

```bash
docker compose -f docker-compose.server.yml down -v
```
